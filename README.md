Данный проект представляет собой прототип системы автоматического обнаружения мошеннических транзакций для финтех-платформы. Он демонстрирует ключевые отличия финтех-разработки от классического IT: встроенные механизмы управления рисками, соответствие регуляторным требованиям и использование машинного обучения для повышения точности детекции.

## Описание проекта

Проект реализует двухуровневую систему фрод-мониторинга:

1. **Правила (rule‑based detection)** – жёсткие логические условия, заданные комплаенс‑ и риск‑менеджерами:
   - контроль суммы транзакции,
   - понижение лимитов для операций из высокорисковых стран,
   - частотный анализ (количество операций пользователя за последний час).

2. **Машинное обучение (ML)** – модель Random Forest, обучаемая на синтетических данных и способная выявлять скрытые закономерности, дополняя правила.

Вся обработка происходит автоматически, каждое решение логируется с указанием причины, что обеспечивает прозрачность для регуляторов и службы поддержки.

## Цели

- **Бизнес‑цель** – продемонстрировать подход к снижению финансовых потерь от мошенничества и соблюдению требований надзорных органов.
- **Техническая цель** – создать расширяемый прототип, который может быть доработан до production‑решения (замена синтетических данных реальными, подключение потоковой обработки и т.д.).

## Запуск

### Локальный запуск

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/kladievaai/IT_proj.git
   cd IT_proj
   ```

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Запустите основной скрипт:
   ```bash
   python crs.py
   ```

   На экран будет выведен отчёт по правилам, метрики качества ML‑модели и значения риска для последних транзакций.

### Автоматическая проверка (GitHub Actions)

Репозиторий настроен на автоматический запуск кода при каждом `push` в любую ветку. Результаты выполнения можно увидеть на вкладке **Actions** вашего репозитория на GitHub. Это позволяет быстро проверять, не сломали ли изменения существующую логику.

## Связь с финтех-спецификой

Проект иллюстрирует основные тезисы из сравнительного анализа [IT vs FinTech](main/analysis.md):

- **Управление рисками** – встроенный риск-контур (лимиты, антифрод, мониторинг).
- **Регуляторное соответствие** – автоматическое применение требований (снижение лимитов для high‑risk стран).
- **Требования к безопасности** – контроль транзакций, трассируемость решений.
- **Операционная готовность** – логирование и аудит.

Ошибка в таком проекте приводит к прямым финансовым и регуляторным последствиям, поэтому код строится вокруг контроля и доказательной базы.

## Результаты и метрики

На синтетических данных модель Random Forest достигает **ROC‑AUC = 0.916**, что говорит о высокой разделяющей способности. Важность признаков показывает, что наибольший вклад вносит сумма транзакции, затем частота операций и час дня. Это соответствует ожиданиям экспертов и подтверждает корректность подхода.
